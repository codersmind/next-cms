## Production

```bash

```

Env: `PORT` (default 3000), `HOSTNAME` (default 0.0.0.0), `JWT_SECRET` (required in production).

### Media uploads (local + S3)

Choose storage via env (`UPLOAD_STORAGE`) or per upload in **Media Library** when both are configured.

| Variable | Purpose |
|----------|---------|
| `UPLOAD_STORAGE` | Default backend: `local` or `s3` |
| `UPLOAD_DIR` | Local folder (default `./uploads`) |
| `UPLOAD_MAX_BYTES` | Max file size (default 10 MB) |
| `AWS_REGION`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `S3_BUCKET` | S3 credentials |
| `S3_PREFIX` | Key prefix in bucket (e.g. `media`) |
| `S3_PUBLIC_BASE_URL` | Optional CDN URL; if omitted, files are proxied at `/api/upload/files/...` |
| `S3_ENDPOINT`, `S3_FORCE_PATH_STYLE` | MinIO / compatible S3 |

`npm start` creates the local upload folder automatically. Role needs `upload.create`.
